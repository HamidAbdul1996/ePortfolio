Clustering with Python

In this unit, we started to use Scikit-learn to employ clustering algorithms and also looked at Principal Component Analysis (PCA). PCA is a dimensionality reduction technique that transforms a dataset into a set of principal components, it's used to simplify complex datasets, improve model performance by eliminating redundant features and help in uncovering patterns in the data that might not be immediately apparent (Chahboun & Maaroufi, 2021).

The elbow method above shows one way to pick the ideal number of clusters. The elbow method calculates the number of clusters by plotting the within-cluster sum of squares (WCSS) against the number of clusters and identifying the point where the decrease in WCSS begins to slow down, resembling an "elbow". In the above picture, the ideal number of clusters is 4. Choosing the ideal number of clusters is important as it improves model performances and ensures that the natural groups within the data are accurately represented (Coraggio & Coretto, 2023).

The code presented above demonstrates how to construct a clustering model and calculate the average values of variables within each cluster.

This example of PCA illustrates its ability to reduce data to two dimensions, enabling us to visualise the data on a scatter plot and clearly identify distinct groups.

Reflection

Reflecting on this seminar, I found it to be fairly straightforward, and I understood everything I completed. An aspect that stood out to me was the discussion on the inapplicability of categorical variables in clustering analyses that use Euclidean distance. The point was interesting, considering that in our Unit 6 assessment, my team and I incorporated categorical variables such as the borough of New York, which we managed through one-hot encoding. At the time, we proceeded thinking this approach would be fine however the insights gained from this seminar and the reading have led me to understand that converting categorical variables into numerical values can inadvertently establish an ordinal relationship when there isn’t one. This also increases the dimensionality of the dataset which can cause challenges. The clusters in our project were not particularly insightful so this could be why. This also led me to look into other distance measures for K-means clustering and the context in which you would implement them. Two others I studied were the Hamming distance and the Jaccard distance. Going forward I will spend more time studying these hyperparameters to better understand the context to use them.
